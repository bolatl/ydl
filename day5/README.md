# Грант-помощник фонда Шахмардана Есенова

Streamlit (и Telegram) RAG-бот на **gemma4**, который отвечает на вопросы о грантах,
программах и стипендиях фонда по данным с сайта
[yessenovfoundation.org](https://yessenovfoundation.org). Отвечает на трёх языках:
русском, казахском и английском.

## Что умеет

- Отвечает на вопросы по грантам и программам фонда (на русском).
- RAG: сначала находит релевантные фрагменты сайта, потом отвечает только по ним.
- **Не выдумывает.** Если в данных нет ответа — честно говорит об этом, а не сочиняет
  дедлайны, суммы и требования.
- Показывает источники (URL страниц) под ответом.
- Помнит контекст диалога в рамках сессии (для follow-up вопросов), но факты всегда
  проверяет через RAG, а не берёт из истории.
- (Опционально) по кнопке отправляет краткое саммари диалога на ваш email через
  MailerSend.
- (Опционально) сообщает контакты фонда, если они есть в данных, и помогает оставить
  заявку: собирает имя + email/телефон + суть запроса и по подтверждению передаёт их
  администратору на email.

## Архитектура

```
scrape.py        -> data/raw_pages.jsonl   (RU-страницы сайта)
build_index.py   -> data/chunks.jsonl + chroma_db/   (нарезка + эмбеддинги)
core/rag.py      -> поиск релевантных фрагментов (единственный источник фактов)
core/llm.py      -> вызов gemma4 + строгий анти-галлюцинационный промпт
app.py           -> Streamlit-чат
```

`core/` не зависит от UI — Telegram-бот (`bot.py`) использует те же функции.

## Языки

В боковой панели (Streamlit) или командой `/lang` (Telegram) выбирается язык ответа:
русский, казахский, английский. Данные с сайта хранятся на русском (на сайте это
параллельные переводы); поиск идёт по русскому корпусу, а ответ бот даёт на выбранном
языке. Факты по-прежнему берутся только из RAG.

## Telegram-бот (опционально)

```bash
python bot.py     # требует TELEGRAM_BOT_TOKEN в .env (получить у @BotFather)
```

Тот же RAG-движок, что и в Streamlit, плюс:
- память диалога в SQLite (`bot_memory.sqlite`), своя на каждый чат, со сжатием старых
  сообщений в краткое саммари;
- `/lang` — переключение языка, `/reset` — очистка контекста, `/start` и `/help`;
- `/send_summary` — отправить саммари администратору (если настроен MailerSend).

## Запуск

```bash
cd day5
pip install -r requirements.txt
cp .env.example .env        # затем впишите выданные на курсе LLM_BASE_URL / LLM_API_KEY /
                            # LLM_MODEL / EMBEDDING_MODEL
python scrape.py            # собрать данные с сайта
python build_index.py       # построить векторный индекс
streamlit run app.py        # запустить чат
```

## Анти-галлюцинации

- `temperature = 0.1`.
- Строгий system-prompt: запрещено выдумывать дедлайны, суммы, требования, документы.
- Если RAG не нашёл релевантных фрагментов (порог `MAX_DISTANCE` в `core/config.py`) —
  бот отвечает «В моих данных нет точной информации по этому вопросу» **без вызова
  модели**.
- Память диалога — только для понимания вопроса, не источник фактов.

## Email-саммари (опционально)

Если в `.env` заданы `MAILERSEND_API_KEY`, `ADMIN_EMAIL` и `MAILERSEND_FROM_EMAIL`,
в боковой панели появляется кнопка «✉️ Отправить саммари администратору». По нажатию
бот делает краткое саммари текущего диалога (gemma4) и отправляет его **только** на
ваш `ADMIN_EMAIL` через MailerSend. Письмо уходит только по явному клику, никогда
автоматически. `MAILERSEND_FROM_EMAIL` должен принадлежать домену, подтверждённому в
MailerSend (на trial-аккаунте это sandbox-домен вида `MS_xxx@trial-yyy.mlsender.net`).

## Заявки и контакты (опционально)

Бот сообщает контакты фонда (email/телефон) только если они есть в собранных данных —
он их не выдумывает. Если пользователь хочет подать заявку или просит связаться с ним,
бот просит указать имя, email/телефон и суть запроса. Когда данные собраны, под чатом
появляется карточка с кнопкой **«✅ Отправить заявку»** — заявка уходит на `ADMIN_EMAIL`
только по явному нажатию (требует настроенного MailerSend, как выше).

## Ограничения

Бот отвечает только по загруженным данным с сайта фонда. Если информации нет, он
сообщает об этом. Это учебный проект (YDL 2026, день 5), а не официальный сервис фонда.

## Настройка качества поиска

После первого `python build_index.py` задайте несколько вопросов и при необходимости
подправьте `MAX_DISTANCE` в [core/config.py](core/config.py): слишком низкий — бот
слишком часто отказывается; слишком высокий — отвечает по нерелевантным фрагментам.
